## Commit Type
- **add** : 새로운 파일, 코드 추가
- **feat** : 새로운 기능 추가, 기존의 기능을 요구 사항에 맞추어 수정
- **fix** : 기능에 대한 버그 수정
- **chore** : 패키지 매니저 수정, 잡일, 그 외 기타 수정 ex) .gitignore
- **docs** : 문서(주석) 수정
- **style** : 코드 스타일, 포맷팅에 대한 수정
- **refactor** : 기능의 변화가 아닌 코드 리팩터링 ex) 변수 이름 변경
- **ui** : 새로운 UI 생성
